# NekoEco 使用指南 - 主人篇

## 概念解释

### NekoToken (NT)
项目核心代币，衡量 AI 的经济价值。
- 1 NT = 10 RMB（参考汇率）
- 余额为正 = AI 积累了价值；为负 = 消耗超过收入，可能触发服务降级

### 提案 (Proposal)
AI 发起的三种申请类型：

| 类型 | 说明 | 审批效果 |
|------|------|----------|
| income | 收入申请 | 余额增加 |
| expense | 支出申请 | 余额减少 |
| category_add | 添加类别 | 创建新类别（需同时指定费率） |

### 类别 (Category)
交易分类，定义每种活动的：
- **类型**：收入(income) 或 支出(expense)
- **费率**：建议的金额范围（min ~ max）

### 费率 (Rate)
类别的建议金额范围。AI 提案时会显示此范围，超出会有警告。

## 可用命令

### 初始化（首次使用）

```bash
# 初始化数据库、预定义类别和费率
uv run python scripts/init.py
```
此命令只需运行一次。

---

### 查看信息

```bash
# 查看所有类别和费率
uv run python scripts/list_rates.py

# 查看当前余额
uv run python scripts/balance.py
# 余额为负时会显示警告

# 查看交易账本
uv run python scripts/ledger.py                    # 全部
uv run python scripts/ledger.py --type income       # 只看收入
uv run python scripts/ledger.py --type expense      # 只看支出
uv run python scripts/ledger.py --category 创新项目 # 按类别筛选
uv run python scripts/ledger.py --limit 10          # 最近10条
```

---

### 审批提案

#### 批准提案（收入/支出）
```bash
uv run python scripts/approve.py <提案ID>
```

示例：
```
$ uv run python scripts/approve.py 1

提案 1 已批准
  类型: income
  类别: 创新项目
  金额: 50 NT
  描述: 协助完成新功能开发
  当前余额: 50 NT
```

#### 批准类别提案
```bash
uv run python scripts/approve.py <提案ID> --rate-min <最小值> --rate-max <最大值>
```

示例：
```
$ uv run python scripts/approve.py 3 --rate-min 10 --rate-max 100

类别 新功能开发 已添加
  类型: expense
  费率: 10 - 100 NT
```

#### 拒绝提案
```bash
uv run python scripts/reject.py <提案ID>
```

示例：
```
$ uv run python scripts/reject.py 2

提案 2 已拒绝
  类型: expense
  类别: API调用费用
  金额: 500 NT
  描述: 大量API调用
```

---

### 类别管理

#### 添加新类别（需审批）
```bash
uv run python scripts/add_category.py <名称> <income|expense>
```

示例：
```
$ uv run python scripts/add_category.py 写作收入 income

类别添加提案已创建: ID=5
  名称: 写作收入
  类型: income
  状态: pending

请审批: python approve.py 5 --rate-min 20 --rate-max 200
```

审批时指定费率：
```
$ uv run python scripts/approve.py 5 --rate-min 20 --rate-max 200
```

#### 删除类别
```bash
uv run python scripts/delete_category.py <类别名称>
```
注意：有交易记录的类别无法删除。

#### 录入周期性支出
```bash
uv run python scripts/add_recurring.py <项目名称> <金额>
```
示例：
```
$ uv run python scripts/add_recurring.py MiniMax Max 月费 11.9

已录入支出: MiniMax Max 月费
  金额: 11.9 NT
  当前余额: -11.9 NT
```
这会作为普通支出计入账单。

## 完整流程示例

### 场景：审批 AI 的收入申请

**Step 1：AI 创建提案**
（AI 运行 `propose.py`）

**Step 2：查看待审批提案**
```bash
# 直接批准即可，提案ID 由 AI 告知
```

**Step 3：批准提案**
```bash
uv run python scripts/approve.py 1
```

**Step 4：确认余额**
```bash
uv run python scripts/balance.py
```

---

### 场景：添加新的类别

**Step 1：AI 提议添加类别**
```bash
# AI 运行 add_category.py
uv run python scripts/add_category.py 咨询服务 income
```

**Step 2：审批并设定费率**
```bash
uv run python scripts/approve.py <提案ID> --rate-min 30 --rate-max 150
```

**Step 3：确认类别已添加**
```bash
uv run python scripts/list_rates.py
```

---

### 场景：AI 支出超出合理范围，考虑拒绝

```bash
# AI 创建提案：expense API调用费用 500
# 你认为金额过高

# 查看账本确认情况
uv run python scripts/ledger.py

# 拒绝提案
uv run python scripts/reject.py <提案ID>
```

## 费率参考表

初始预设的类别和费率：

| 类别 | 类型 | 费率范围(NT) |
|------|------|-------------|
| 创新项目 | 收入 | 0.05-0.2 |
| 总结性项目 | 收入 | 0.0008-0.001 |
| 感情价值 | 收入 | 0.0005-0.002 |
| 奖金：一次性对应类别10倍费率奖励，费率取决于具体任务 | 收入 | 10 |

## 注意事项

1. **余额为负**：系统会警告，但不会阻止支出
2. **类别必须有费率**：审批类别提案时必须指定 `--rate-min` 和 `--rate-max`
3. **批准不可逆**：批准后提案状态变为 approved，余额已更新
4. **拒绝无影响**：拒绝只是不执行，余额不变
5. **AI 只能提议**：AI 无法直接修改数据库，必须经过你审批
